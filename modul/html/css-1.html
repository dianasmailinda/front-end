  • In-line Styles

    Styles are applied directly to a particular html element

Selectors

  • Basic
    • Type

      e.g. body element the selector-> body{}

    • Identifier

      e.g. id="username" the selector-> #username{}

    • Class

      e.g. class="black" the selector-> .black{}

    • Attribute

      e.g. Google Instagram (Check the code)

  • Relational

    • The selector select element based on the relationship with other elements

    These selectors help us write cleaner markup, but..

    • they can be fragile, bc they're dependent on the location of these elements in the DOM. so if the element is moved around, the styles would break.
    • they are not as fast as basic selectors, you can barely see it, but they are indeed lack of performance since they would need to read more lines of code than necessary.

    Relational selectors:

    • " "

      s1 s2{}

      selects all the s2 inside the s1

    • >

      s1 > s2{}

      selects the first s2 inside s1 (s2 is direct child of s1)

    • +

      s1 + s2{}

      selects the first s2 that comes immediately after s1 (s2 is direct sibling of s1)

    • ~

      s1 ~ s2{}

      selects all s2 that comes after s1 (s2 are sibling of s1)

    e.g.

    Lorem ipsum dolor sit amet.

    Lorem ipsum dolor sit amet consectetur adipisicing elit. Eligendi, rem.

    Lorem ipsum dolor sit amet consectetur adipisicing elit. Odio, tempora.

    Lorem ipsum dolor sit amet consectetur adipisicing elit. Odio, tempora.

Note:

  • 1 id can only be used by 1 element
  • 1 class can be used multiple time by multiple elements
